# DC 悬浮记事本插件 - 方案确认书

## 一、痛点分析

| 痛点 | 场景 | 后果 |
|------|------|------|
| Todo 散落 | 单点会议 todo 顺手打在 DC 聊天框 | 遗忘、难追溯 |
| 临时信息无归处 | 阅后即焚的 API Key、临时密码 | 过期丢失、无法找回 |
| 截图无序堆积 | 会议中截图、培训截图堆在聊天记录 | 翻找困难 |
| 碎片想法易丢 | 培训疑问、灵光一现的想法 | QA 环节想不起来 |
| 聊天框杂乱 | 以上内容全部混在聊天流中 | 信息检索成本极高 |

**核心矛盾**：DC 是「沟通流」，不是「记录池」——但大家习惯用 DC 做临时记录，因为它是始终打开的窗口。

---

## 二、产品定位

> **一个悬浮在 DC 左上角的轻量记事本，让"随手记"不再污染聊天框，记完还能自动归类、快速找回。**

**一句话**：DC 旁边的「第二大脑草稿纸」。

---

## 三、功能设计

### 3.1 核心功能

```
┌─────────────────────────────────────────┐
│  📌 DC 悬浮记事本              ─ □ ✕   │
├─────────────────────────────────────────┤
│  🔍 搜索...                   🎤 📷 📋 │
├─────────────────────────────────────────┤
│  📁 待办        📁 灵感    📁 临时信息  │
│  ├ 完成需求评审   ├ 定价策略  ├ API Key   │
│  ├ 发送周报       └ ...      └ 临时密码  │
│  └ ...                     └ ...        │
├─────────────────────────────────────────┤
│  💬 输入文字/粘贴截图/录音...            │
└─────────────────────────────────────────┘
```

#### 📝 快速录入（3 秒内完成记录）

| 录入方式 | 操作 | 示例 |
|----------|------|------|
| **文字** | 直接输入，回车保存 | "下午3点跟张三对齐方案" |
| **截图** | Ctrl+V 粘贴 / 拖拽 / 点击📷按钮 | 会议 PPT 截图 |
| **语音** | 点击🎤按钮，说话自动转文字 | "这个接口需要加限流" |
| **剪贴板** | 点击📋按钮，一键抓取当前剪贴板内容 | 复制的 API Key |

#### 🏷️ 自动归类

采用 **AI 自动分类 + 手动调整** 双轨模式：

| 类别 | 自动识别规则 | 图标 |
|------|-------------|------|
| **待办 Todo** | 含"要做/完成/跟进/提醒/记得"等关键词 | ✅ |
| **临时信息** | 含 key/token/密码/账号/链接 等特征 | 🔑 |
| **会议笔记** | 含"会议/讨论/对齐/评审"等 + 截图 | 📋 |
| **灵感想法** | 含"想法/试试/能不能/如果"等 | 💡 |
| **疑问** | 含"为什么/怎么/？/不懂"等 | ❓ |
| **其他** | 无法自动归类时归入 | 📌 |

- 每条记录自动打标签 + 时间戳
- 支持用户手动拖拽调整分类
- 支持 # 自定义标签（如 `#sprint12` `#定价项目`）

#### 🔍 快速找回

| 方式 | 说明 |
|------|------|
| **全文搜索** | 搜索框实时检索标题、内容、标签 |
| **分类筛选** | 点击左侧分类快速过滤 |
| **时间线** | 按时间倒序浏览，支持日期跳转 |
| **标签聚合** | 点击标签查看同类所有记录 |

### 3.2 增强功能

| 功能 | 说明 |
|------|------|
| **⏰ 提醒** | 对 Todo 设置提醒时间，到时 DC 消息通知 |
| **📌 置顶** | 重要信息置顶，如 API Key、当天待办 |
| **📤 一键转 Todo** | 记录的 Todo 可一键同步到 Smartwork 待办 |
| **📤 一键发消息** | 整理好的内容一键发送到指定 DC 会话 |
| **🔒 敏感信息标记** | API Key 等自动标记，支持过期提醒/手动销毁 |
| **🎨 Markdown 支持** | 文字内容支持 Markdown 格式化 |

## 四、技术方案

### 4.1 形态选择

> **推荐方案：Chrome Extension（浏览器扩展） + DC Web 版注入**

| 方案 | 可行性 | 悬浮效果 | 分发难度 | 开发周期 |
|------|--------|---------|---------|---------|
| ❌ DC 原生插件 | 需要DC平台支持 | ✅ 完美 | 中 | 需协调 |
| ✅ **Chrome Extension** | **完全可行** | **✅ 完美** | **低** | **2周** |
| ⚠️ 独立 Electron 窗口 | 可行 | 需手动对齐 | 高 | 3周 |
| ⚠️ 独立 Web 页面 | 可行 | ❌ 无法悬浮 | 低 | 1周 |

**选择 Chrome Extension 的理由**：
1. DC Web 版在浏览器中使用，Extension 可直接注入悬浮 UI
2. 可通过 `content_scripts` 在 DC 页面上注入浮动面板
3. 支持键盘快捷键全局唤起（如 `Alt+N`）
4. 本地 `chrome.storage` 存储数据，无需额外后端
5. 可利用 `chrome.clipboard` API 读取剪贴板
6. 可调用 Web Speech API 实现语音转文字
7. 打包为 `.crx` 文件，同事安装即可使用

### 4.2 架构设计

```
┌─────────────────────────────────────────────────┐
│                Chrome Extension                  │
├──────────────┬──────────────┬────────────────────┤
│  Background  │  Content     │  Popup /           │
│  Service     │  Script      │  Side Panel        │
│  Worker      │  (注入DC页面) │  (扩展弹窗)        │
├──────────────┼──────────────┼────────────────────┤
│ - 快捷键监听  │ - 悬浮面板UI │ - 设置面板         │
│ - 剪贴板读取  │ - 截图粘贴   │ - 分类管理         │
│ - AI分类调用  │ - 语音录入   │ - 数据导入/导出    │
│ - 定时提醒    │ - 内容渲染   │ - 同步配置         │
├──────────────┴──────────────┴────────────────────┤
│              chrome.storage.local                │
│         (本地持久化，数据不出本机)                  │
├─────────────────────────────────────────────────┤
│              AI 分类服务 (可选)                   │
│     本地规则引擎 (默认) / LLM API (增强)          │
└─────────────────────────────────────────────────┘
```

### 4.3 数据模型

```json
{
  "id": "uuid",
  "type": "todo|info|meeting|idea|question|other",
  "title": "自动提取的标题",
  "content": "正文/Markdown/图片Base64",
  "contentType": "text|image|voice|mixed",
  "voiceUrl": "录音Blob URL",
  "tags": ["#sprint12", "#定价"],
  "source": "manual|clipboard|voice|screenshot",
  "priority": "high|medium|low",
  "reminder": "2026-05-24T15:00:00+08:00",
  "pinned": false,
  "sensitive": false,
  "expireAt": "2026-05-25T00:00:00+08:00",
  "createdAt": "2026-05-23T10:30:00+08:00",
  "updatedAt": "2026-05-23T10:30:00+08:00"
}
```

### 4.4 AI 自动分类策略

**第一层：本地规则引擎（离线可用，零延迟）**

```javascript
// 关键词匹配 + 优先级规则
const rules = {
  todo:    /要做|完成|跟进|提醒|记得|待办|TODO/i,
  info:    /key|token|密码|账号|密码|api[_-]?key|secret/i,
  meeting: /会议|讨论|对齐|评审|周会|站会|sync/i,
  idea:    /想法|试试|能不能|如果|也许|可以考虑/i,
  question:/为什么|怎么|如何|\?|？|不懂|疑问/i,
};
```

**第二层：LLM 增强分类（可选，更智能）**

- 当本地规则置信度低时，调用 LLM API 做意图识别
- 用户可配置是否启用（默认关闭，保护隐私）
- 支持自定义分类规则（如"所有含项目名的归为一类"）

### 4.5 关键交互流程

**快速记录流程**：
```
用户按 Alt+N / 点击悬浮图标
  → 面板展开（动画 200ms）
  → 输入框自动聚焦
  → 输入文字 / Ctrl+V粘贴截图 / 点🎤录音
  → 回车/点击保存
  → AI 自动分类 + 打标签
  → 面板折叠，Toast 提示"已保存到 ✅待办"
```

**信息找回流程**：
```
用户点击悬浮图标
  → 面板展开，展示分类视图
  → 搜索框输入关键词 / 点击分类Tab
  → 命中结果高亮展示
  → 点击条目展开详情
  → 可编辑/删除/置顶/设置提醒
```

## 七、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| DC Web 版 DOM 变更导致注入失败 | 面板无法显示 | 使用 Shadow DOM 隔离，降低耦合 |
| 敏感信息存储在本地 | 安全风险 | 支持"阅后即焚"自动清理 + 加密存储 |
| Chrome Extension 安装门槛 | 同事不愿装 | 提供独立 Web 版作为降级方案 |
| 语音识别精度 | 体验差 | 优先支持文字+截图，语音为辅助 |

## 八、降级方案

如果 Chrome Extension 方案遇到障碍，可退化为：

1. **独立 Web 页面**：部署到 OE，书签打开，失去悬浮能力但功能完整
2. **DC Bot 方式**：在 DC 中 @机器人 记录，机器人自动归类并回复分类结果
3. **桌面 Widget**：Electron 小窗口，始终置顶，可悬浮在 DC 旁边
